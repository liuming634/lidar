> 东北大学T-DT实验室 RoboMaster 2025赛季 雷达站软件系统

## 1. 整体架构概览

本项目是一个**多传感器后融合的雷达站系统**，核心设计理念是**传感器完全解耦**——激光雷达（Livox Avia）和单目相机（海康威视）各自由独立的管线处理，在**状态层面（卡尔曼滤波）** 而非原始数据层面融合，避免了联合标定的误差，且开发难度不随传感器数量增加。

## 3. 各部分实现思路详解

### 3.1 LiDAR数据定位 —— localization 节点

**输入**: `/livox/lidar` (Livox Avia 10Hz 原始点云)  
**输出**: TF变换 `rm_frame → livox_frame`、`/livox/map` (场地地图可视化)

**核心方法: GICP（广义迭代最近点）配准**

1. **加载场地先验地图**: 读取 `config/RM2025.pcd` 作为GICP的`target`点云
2. **多帧累积**:
   - 维护FIFO缓冲区，累积20帧实时点云（`accumulated_clouds_`）
   - 多点云累积增加密度，提高配准稳定性
3. **球面栅格滤波去噪**:
   - 按方位角(azimuth)和俯仰角(elevation)以0.1°步长划分球面栅格
   - 每个栅格只保留**距离最远**的点（消除近距离噪点和激光雷达自身的散射点）
   - 原理: Livox Avia是非重复扫描，近距离会有大量噪点，取最远点可以保留真实物体点
4. **空间范围裁剪**: 只保留 `x:5~30, y:-10~8, z<7` 范围内的点
5. **体素下采样**: `VoxelGrid` 叶大小 `0.1m`，降低计算量
6. **GICP配准**:
   - 将实时点云(source)与场地地图(target)做GICP配准
   - 配准分数(fitness score) `< 0.2` 认为收敛，标记 `has_aligned_ = true`
   - 从4×4变换矩阵提取平移和旋转 → 四元数 → 广播TF
7. **周期发布**: 每10秒在 `/livox/map` 发布一次场地点云供可视化

**为什么用GICP而不是普通ICP**:
GICP比ICP对点云密度变化更鲁棒，Livox Avia的非重复扫描特性导致每帧点云分布不同，GICP能更好地处理这种变化。

---

### 3.2 LiDAR动态物体检测 —— dynamic_cloud 节点

**输入**: `/livox/lidar`、TF(`rm_frame→livox_frame`)  
**输出**: `/livox/lidar_dynamic` (动态点云)、`/livox/lidar_other` (其他区域点云)、`/lidar_detect` (预警状态)

**核心方法: KD-tree最近邻查询的动态点判定 + 空间规则预警**

#### 动态点提取流程

1. **坐标变换**:
   - 通过`tf_buffer_.lookupTransform`获取 `rm_frame→livox_frame` 的变换
   - 构建 `Eigen::Affine3f`，对所有点逐点做仿射变换，将雷达点云转到场地坐标系

2. **空间范围过滤**:
   - 只保留场地内有效点: `x:3~28, y:0~15, z:0~1.4`
   - 过滤掉对角线区域等无效范围
   - 同时将**非场地但需要监控的区域**（飞镖区、无人机通道）分离到 `other_filtered_cloud`

3. **KD-tree动态点判定**:
   - 将 `map_cloud`（RM2025.pcd 体素下采样后）构建KD-tree
   - 对每个实时点，在KD-tree中查询最近邻（`nearestKSearch`, K=1）
   - 如果最近邻平方距离 > 阈值(0.01 = 0.1m²)，判定为动态点
   - **多线程加速**: 12线程并行处理，按点数均分任务

4. **多帧FIFO累积**:
   - 维护3帧的FIFO队列 `accumulated_clouds_`
   - 多帧累积增强稳定性，消除单帧误检

#### 预警检测（特殊区域规则）

5. **飞镖检测** (`dart_cloud_filter`):
   - 定义敌方基地前的空间立方体区域（x:27.22~27.41, y:3.925~4.525, z:1.713~2.472）
   - 落入该区域的点数 > 5，则 `dart_state = 1`（有飞镖）

6. **无人机三级预警**:
   - **安全区** (fly_safe): 己方半场上空起飞区，点数 >40 → 状态1
   - **预警区** (fly_warn): 己方半场靠近基地，点数 >40 → 状态2  
   - **报警区** (fly_alarm): 场地中场，点数 >40 → 状态3（最高威胁）
   - 三个区域按空间位置从近到远划分，实现递进预警

### 3.3 LiDAR目标聚类 —— cluster 节点

**输入**: `/livox/lidar_dynamic` (动态点云)  
**输出**: `/livox/lidar_cluster` (聚类质心点云)

**核心方法: PCL欧几里得聚类**

1. 将动态点云从 `sensor_msgs/PointCloud2` 转为 `pcl::PointCloud<pcl::PointXYZ>`
2. 为点云构建KD-tree（加速邻域搜索）
3. 创建 `pcl::EuclideanClusterExtraction` 提取器:
   - `setClusterTolerance(0.25)`: 同簇内点距离不超过0.25m
   - `setMinClusterSize(5)`: 簇最少5个点（过滤过小噪声簇）
   - `setMaxClusterSize(1000)`: 簇最多1000个点
4. 对每个提取到的簇，计算所有点XYZ坐标的**均值**作为该目标的质心
5. 将所有质心点合并为一个点云发布

**为什么欧几里得聚类**:
RoboMaster场地中的机器人（步兵、英雄等）尺寸大约0.5~1m，0.25m的容差可以有效区分不同的机器人，同时将同一个机器人的点聚到一起。

### 3.4 相机数据 —— hik_camera 节点

**输入**: 海康威视工业相机(CH-120-10UC) 硬件  
**输出**: `/camera_image` (sensor_msgs/Image, bgr8)

**核心方法: 海康MVS SDK采集 + Bayer格式转换**

1. 初始化海康MVS SDK (`MV_CC_Initialize`)
2. 枚举USB相机设备，取第一个可用相机
3. 创建句柄 → 打开设备 → 设置连续采集模式(TriggerMode=0)
4. 开始取流(`MV_CC_StartGrabbing`)
5. 33ms定时器循环取帧:
   - `MV_CC_GetImageBuffer` 获取原始帧
   - 判断像素格式: 如果是Bayer格式(BayerRG8/GB8/GR8/BG8)，用OpenCV `cv::cvtColor` 转为BGR
   - 构造 `sensor_msgs::msg::Image` 发布到 `/camera_image`
6. 析构时释放资源: 停止取流 → 关闭设备 → 销毁句柄 → SDK反初始化

**相机数据用途**: 作为视觉管线的图像输入，经过三级级联检测识别出敌方机器人的像素位置，再通过透视变换转换为场地坐标，用于给卡尔曼滤波提供颜色和编号信息。

### 3.5 三级级联检测 —— detect 节点

**输入**: `/camera_image` (sensor_msgs/Image, bgr8)  
**输出**: `detect_result` (vision_interface/DetectResult)

**核心方法: YOLOv8(车辆) → YOLOv8(装甲板) → DenseNet121(数字) 三级级联**

这是本项目最核心的视觉创新点，与传统单阶段检测不同:

#### 第一级: YOLOv8s 车辆检测 (1280×1280)
- 输入: 完整相机图像
- 输出: 车辆检测框（class_label=0/1, 最多12辆）
- 置信度阈值: 0.65, NMS阈值: 0.45
- 筛选出所有机器人框

#### 第二级: YOLOv8s 装甲板检测 (192×192)
- 对每个车辆框裁剪ROI区域
- 批量送入 `armor_yolo` 检测装甲板（最多20个装甲板）
- 置信度阈值: 0.4, NMS阈值: 0.45
- 192×192 的输入尺寸专门针对装甲板的大小优化

#### 第三级: DenseNet121 数字分类 (224×224)
- 对每个装甲板裁剪ROI
- 批量送入 `classifier` 识别数字编号(1~5)
- 通过 `classifier->forwards(armor_images)` 批量推理

#### 颜色判断
- 对每个装甲板ROI，拆分BGR三通道
- 计算 `B-R` 和 `R-B` 通道差的均值
- 若 `avg(B-R) > avg(R-B)` → 蓝色(0)，否则红色(2)
- 原理: 蓝色装甲板的灯条反蓝光，B通道值高；红色同理

#### 结果组装
- 对每辆车，选置信度最高的有效装甲板（编号不为0和5）
- 填入 `DetectResult` 消息的对应编号槽位（1~5号，index从0开始）
- 6号预留为空

#### 工程细节
- 自动ONNX→TensorRT引擎转换: 检测 `.engine` 文件不存在时自动调用 `onnx2trt.py`
- 支持按 `r` 键切换debug可视化
- RTX A4000实测50Hz推理速度

**为什么三级级联**:
1. 单阶段直接检测小目标（装甲板、数字）需要很高的输入分辨率，计算量大
2. 先检车辆 → 裁剪 → 检装甲板 → 裁剪 → 分类，每级处理更小的图像，整体更高效
3. 每级模型可独立训练和调优，降低数据集制作难度
4. 级联结构天然具有鲁棒性: 即使车辆检测框有偏移，装甲板检测仍可能正确

### 3.6 坐标解算 —— resolve 节点

**输入**: `detect_result` (2D像素坐标)  
**输出**: `resolve_result` (场地3D坐标)、`camera_point3D` (彩色点云)

**核心方法: 基于区域高度的透视变换**

#### parser类（radar_utils）核心算法

1. **加载标定参数**:
   - 相机内参: `config/camera_params.yaml` (焦距3617, 主点2048×1515)
   - 相机外参: `config/out_matrix.yaml` (世界坐标系旋转向量+平移向量)

2. **基于区域的动态高度**:
   - 场地划分为多个多边形区域（中路、左路、右路、敌方增益区、己方要塞、敌方要塞）
   - 每个区域有预设高度（0.15~0.6m）
   - 通过 `cv::pointPolygonTest` 判断2D点落在哪个区域
   - 加载 `config/RM2025_Points.yaml` 中的多边形顶点

3. **透视变换解算** (`get_2d`):
   - 选择4个3D参考点（固定世界坐标，高度 = ARMOR_HEIGHT(0.15) + 区域高度）
   - 用 `cv::projectPoints` 将3D参考点投影到图像平面（利用PNP外参）
   - 得到4组图像2D↔场地2D对应点对
   - 用 `cv::getPerspectiveTransform` 计算透视变换矩阵
   - 用 `cv::perspectiveTransform` 将输入图像点映射到场地坐标

4. **坐标处理**:
   - 蓝色系: `y = 15 + center_point.y`（映射到场地右半区）
   - 红色系: 保持y不变
   - 发布彩色点云（蓝方蓝色点、红方红色点）

**为什么透视变换而不是直接PNP**:
透视变换可以从2D直接映射到2D（图像平面到场地平面），计算简单且实时性好。配合动态高度调节，可以适应场地不同区域的高度变化（如坡道、增益区）。

### 3.7 相机标定 —— calibrate 节点

**输入**: 鼠标点击的用户交互  
**输出**: `config/out_matrix.yaml` (外参)

**核心方法: 手动5点PNP标定**

1. 加载相机内参，订阅 `/camera_image` 或从rosbag读取
2. 用户按Enter进入标定模式
3. 鼠标在图像上点击场地特征点（预知3D坐标）:
   - 点1: 我方堡垒靠近我方的底角
   - 点2: 我方前哨站血条底部
   - 点3: 敌方基地引导灯
   - 点4: 敌方前哨站引导灯
   - 点5: 敌方四十三度坡围挡左底角
4. 每个点用 `WASD` 键微调像素位置，`n` 键确认
5. 5个点确认后，用 `cv::solvePnP`（EPNP算法）求解外参
6. 结果保存到 `config/out_matrix.yaml`

**为什么5个点**: 3个点可解（理论上PNP最少3点），但5个点提供了冗余，提高标定精度和鲁棒性。选择的5个特征点覆盖场地不同区域（近处、远处、高处），使标定结果分布更均匀。

### 3.8 多目标卡尔曼滤波融合 —— kalman_filter 节点

**输入**: `/livox/lidar_cluster` (LiDAR聚类)、`resolve_result` (相机解算)、`/lidar_detect` (预警)、`/match_info` (比赛信息)  
**输出**: `/kalman_detect`、`/radar2sentry`、`/livox/lidar_kalman`

**核心方法: 多个CV(Constant Velocity)卡尔曼滤波器 + LiDAR主时钟 + 相机辅助匹配**

> ⚠️ **关键理解: 整个系统只有一个卡尔曼滤波节点，不存在"两个滤波器结果融合"**  
> LiDAR 负责驱动KF的状态更新（"目标在哪里"），相机负责提供颜色和编号信息（"目标是什么"）。  
> 两种传感器的数据在**同一个KF内部**的`detect_history`中汇合，而非两个独立的KF输出再合并。

#### 融合架构总览

```
                          ┌──────────────────────────────────────┐
LiDAR聚类点 ─────────────►│  kalman_filter_node                  │
  /livox/lidar_cluster    │                                      │
       (主时钟,10Hz)       │  ┌──────────┐   ┌───────────────┐   │
                          │  │ predict()├──►│  LiDAR点匹配   │   │
                          │  │ (全部KF) │   │  match+update  │   │
                          │  └──────────┘   └───────┬───────┘   │
                          │                          │           │
                          │  ┌───────────────────────▼───────┐   │
相机检测点 ─────────────►│  │  camera_match()              │   │
  /resolve_result         │  │  在LiDAR历史轨迹中找最近点     │   │
      (辅助,~50Hz)        │  │  匹配成功→记录(color,number) │   │
                          │  └───────────────────────────────┘   │
                          │                         │           │
                          │                         ▼           │
                          │  ┌───────────────────────────────┐   │
                          │  │  投票决定颜色和编号            │   │
                          │  │  get_color() → 蓝/红          │   │
                          │  │  get_number() → 1~5号         │   │
                          │  └───────────────────────────────┘   │
                          └──────────────────────────────────────┘
```

**总结一句话: LiDAR 负责"在哪里"（位置跟踪），相机负责"是什么"（颜色+编号），融合是在单个多目标KF内部完成的，不是两个KF结果合并。**

#### Kalman_filter_plus 单目标滤波器

**状态模型 (CV模型)**:
```
状态向量: x = [x, vx, y, vy]^T   (位置 + 速度)
观测向量: z = [x, y]^T            (仅位置)
转移方程: x_k = F * x_{k-1}       F = [[1, dt, 0, 0],
                                       [0, 1,  0, 0],
                                       [0, 0, 1, dt],
                                       [0, 0, 0, 1 ]]
观测方程: z_k = H * x_k            H = [[1, 0, 0, 0],
                                       [0, 0, 1, 0]]
```

**过程噪声协方差 Q**: 由 `sigma_q_x=50.0` 和 `sigma_q_y=50.0` 以及 `dt` 动态计算  
**观测噪声协方差 R**: 固定对角阵 `diag(0.1, 0.1)`

#### 多目标融合策略

**LiDAR主时钟（由/livox/lidar_cluster订阅触发）**:

1. **Predict阶段**: 对所有KF执行 `update_predict_point()`:
   - 用 `get_time()` 获取自上次更新以来的实际时间间隔 `dt`
   - 执行 `KF.predict()` 更新预测位置
   - 重置 `has_updated = false`

2. **Match阶段**: 对每个LiDAR聚类点，遍历所有KF做距离匹配:
   - 匹配条件: `Distance(predict_point, input) < car_max_speed * dt + detect_r`
     - `car_max_speed = 2.5m/s`（机器人最大速度）
     - `detect_r = 1.0m`（检测半径）
   - 无匹配 → 新建KF
   - 唯一匹配 → 执行 `KF.update()`（`KF.correct()` + 记录history）
   - 多匹配 → 选最近的点更新

3. **Cleanup阶段**:
   - 删除 `last_time > 1.5s` 的KF（目标丢失超时）
   - 存活KF通过 `detect_history` 投票决定颜色和编号

4. **坐标翻转**: 蓝方自方时，对坐标做 `(28-x, 15-y)` 翻转（统一到以蓝方基地为原点）

**相机辅助匹配** (`camera_match` 方法):
- 相机检测结果到达频率比LiDAR低
- 在KF的 `history`（LiDAR历史轨迹）中找时间最近的点
- 如果时间差 < 1.0s 且空间距离 < detect_r，则记录 `(color, number)` 到 `detect_history`
- 通过 `get_color()` / `get_number()` 对历史记录统计投票

**为什么CV模型**:
RoboMaster比赛中的机器人运动虽然复杂，但在短时间间隔内（0.1s）可以近似为匀速运动。CV模型简单可靠，计算量小，且性能已经足够。

**为什么LiDAR做主时钟**:
LiDAR的频率稳定（10Hz），且能直接提供3D位置，是天然的跟踪主时钟。相机提供颜色和编号信息（这些都是LiDAR无法直接获得的），作为辅助信息匹配到LiDAR轨迹上。

### 3.9 决策输出 —— debug_map 节点

**输入**: `/kalman_detect` (融合结果)、`resolve_result` (相机直接结果)、`/match_info`  
**输出**: `/Radar2Sentry`、`/hero_state` (英雄状态)、`/map_2d` (可视化图像)

**核心方法: 2D小地图可视化 + 新鲜度控制 + 英雄进退场检测**

1. **小地图绘制**:
   - 将 `config/RM2025.png` 加载为底图（缩放至28×15比例）
   - 在对应坐标绘制蓝/红方圆点+编号
   - 0.5秒时间窗新鲜度控制（超时的不显示）

2. **雷达→哨兵数据发送** (`Radar2Sentry`):
   - 基于比赛标记分数(`marks`)控制发送策略:
     - `marks >= 117`: 标记满，进入放松状态，降低发送频率（每0.35s发一次）
     - `marks < 105`: 标记不足，恢复正常发送
     - `marks` 在105~117之间: 不发送，等待标记恢复
   - **核心优化**: 减少不必要的数据发送，只在数据有效时发送，避免浪费通信带宽

3. **英雄机器人进退场检测** (`hero_state`):
   - 通过机器人id=0的x坐标判断英雄位置
   - x坐标在基地附近 → hero_count1++ → 状态1（在基地/准备出场）
   - x坐标在前哨站/增益区 → hero_count2++ → 状态2（在场上前进）
   - 计数 > 10 时触发状态发布

### 3.11 基于PCD→PNG映射的场地高程分析（新方案）

**思路**: 将聚类提取后的动态物体代表点XY坐标映射到高程PNG图像上，取对应区域的有效灰度数据做平均，反算出高度，并用配置文件做范围矫正。

#### 核心数据流

```
LiDAR点云 → 欧几里得聚类 → 提取每个物体的代表点 (x, y, z)
                                          │
                                          ▼
                              代表点的XY坐标映射到PNG像素位置 (u, v)
                                          │
                                          ▼
                       在PNG上取 (u, v) 周围一定区域的像素灰度值
                                          │
                                          ▼
                       收集区域内所有有效灰度值 (G ≠ 255)
                       计算平均灰度值 G_avg
                                          │
                                          ▼
                       通过公式反算高度: H = G_avg × 0.008490 - 0.187180
                                          │
                                          ▼
                       用PCD点云的真实 z 值对推算的 H 做矫正
```

#### 灰度-高度映射公式

**基本信息**:
| 项目 | 数值 |
|------|------|
| 图像来源 | 2.tiff 高程图 |
| 图像尺寸 | 2916 × 1505 像素 |
| 灰度位深 | 8 bit (0~255) |
| 最低高度 | -0.187180 m |
| 最高高度 | 1.969253 m |
| 每级灰度 | 8.49 mm/级 |

**计算公式**:

灰度值 G (0 ≤ G ≤ 254) → 高度 H：
```
H = G / 254 × (1.969253 - (-0.187180)) + (-0.187180)
H = G / 254 × 2.156433 - 0.187180
H = G × 0.008490 - 0.187180
```

高度 H → 灰度值 G：
```
G = (H + 0.187180) / 2.156433 × 254
G = (H + 0.187180) / 0.008490
```

**精度**: 每级灰度对应 8.49mm，灰度 255 表示无数据。

#### 实现方式

**1. 聚类代表点提取**
- LiDAR动态点云经欧几里得聚类后，每个cluster计算质心 `(x, y, z)` 作为代表点
- 代表点的 (x, y) 用于映射到PNG图像，(z) 作为真实高度用于后续矫正

**2. XY→UV坐标映射**
- 将场地坐标系下的 (x, y) 转换到PNG图像的像素坐标 (u, v)
- 需要知道PNG的分辨率（米/像素）和原点对应的像素位置

**3. 局部区域灰度采样**
- 在 (u, v) 周围取一个窗口（如 5×5 或 3×3 像素）
- 收集窗口内所有灰度值 G ≠ 255 的有效像素
- 计算有效像素的平均灰度值 G_avg

**4. 高度反算与矫正**
- 用公式 `H = G_avg × 0.008490 - 0.187180` 反算高度
- 用聚类点的真实 z 值对比反算结果，建立矫正参数
- 矫正参数保存到配置文件中，后续直接用于高度推算

#### 高度矫正

```
对于第 i 个聚类物体:
  真实高度 z_i (来自PCD点云)
  灰度推算高度 H_i (来自PNG灰度平均)

建立多组对应关系后:
  计算偏差: offset = mean(z_i - H_i)
  矫正后高度: H_corrected = H + offset
```

当场地环境变化时，仅需调整配置文件中的高度范围参数或矫正偏移量即可。

#### 代码示例

```python
import cv2
import numpy as np

# 加载灰度高程图
img = cv2.imread('config/2_elevation_gray256.png', cv2.IMREAD_GRAYSCALE)

# 聚类代表点 (x, y) → 像素坐标 (u, v)  （需根据实际分辨率换算）
def xy_to_uv(x, y, resolution, origin_x, origin_y):
    u = int((x - origin_x) / resolution)
    v = int((origin_y - y) / resolution)  # y轴翻转
    return u, v

# 在 (u,v) 周围取窗口平均灰度
def get_avg_height(img, u, v, window_size=5):
    half = window_size // 2
    roi = img[max(0, v-half):v+half+1, max(0, u-half):u+half+1]
    valid = roi[roi != 255].astype(np.float32)
    if len(valid) == 0:
        return None  # 无有效数据
    G_avg = np.mean(valid)
    H = G_avg * 0.008490 - 0.187180
    return H

# 对每个聚类代表点查询高度
for cluster_x, cluster_y in representative_points:
    u, v = xy_to_uv(cluster_x, cluster_y, res, origin_x, origin_y)
    H = get_avg_height(img, u, v)
    print(f"({cluster_x:.3f}, {cluster_y:.3f}) → 高度: {H:.4f}m")
```

## 4. 核心设计亮点总结

| 设计 | 解决的问题 | 实现方法 |
|------|-----------|---------|
| **传感器完全解耦** | 避免联合标定误差、降低复杂度 | 各自独立管线，状态层面后融合 |
| **三级级联神经网络** | 高精度检测小目标、降低数据集难度 | YOLOv8(整图)→YOLOv8(ROI)→DenseNet(ROI) |
| **GICP自动配准** | 无需人工初始化，即插即用 | 球面栅格滤波+GICP配准+场地PCD |
| **基于区域高度透视变换** | 适应场地不同高度区域 | 多边形区域划分+动态高度+perspectiveTransform |
| **多目标CV卡尔曼滤波** | 融合LiDAR位置+相机颜色编号（并非两个滤波器结果合并） | LiDAR主时钟(predict/update) + 相机辅助(camera_match贴颜色编号标签) + 历史投票 |
| **球面栅格滤波** | Livox非重复扫描的噪点过滤 | 0.1°方位角/俯仰角栅格取最远点 |
| **飞镖/无人机空间规则** | 无需额外传感器即可预警 | 空间立方体区域+点数阈值判定 |
