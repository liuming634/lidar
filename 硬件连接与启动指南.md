## 二、激光雷达（Livox Avia）

### 2.1 修改广播码

每个 Livox 雷达机身贴纸上有唯一的广播码。将驱动中所有 `cmdline_bd_code` 和 `broadcast_code` 改为雷达的广播码。

修改 `src/livox_driver/livox_ros2_driver/launch/livox_lidar_launch.py` 第 16 行：

```python
cmdline_bd_code = 'livox你的广播码'   # 改为雷达机身上的广播码
```

同样修改 `src/livox_driver/livox_ros2_driver/config/livox_lidar_config.json` 第 4 行的广播码，并确认 `enable_connect` 为 `true`：

```json
"broadcast_code": "livox你的广播码",
"enable_connect": true,
```

> 两个文件中的广播码必须一致，否则雷达连接不上。驱动中其他 launch 文件（`livox_template_launch.py`、`livox_lidar_msg_launch.py` 等）也建议一并修改。

### 2.2 接线与网络

雷达用网线直连电脑（或通过交换机）。

你的有线网卡名是 `enp3s0`，推荐以下设置：

```
电脑 IP：192.168.1.100（静态）
子网掩码：255.255.255.0
网关：192.168.1.1
```

设置方法（桌面版 Ubuntu — **设置 → 网络 → 有线 → 齿轮图标 → IPv4 → 手动**，填入上面三个值；或用命令 `nmcli con mod enp3s0 ipv4.addresses 192.168.1.100/24 ipv4.method manual`）。

> 网线插上后用 `ip a` 确认 `enp3s0` 状态从 `NO-CARRIER` 变为 `UP`，且有配置的 IP 地址。

### 2.3 启动雷达

```bash
ros2 launch livox_ros2_driver livox_lidar_launch.py
```

连接成功会看到以下输出（关键信息）：

```
[livox_ros2_driver_node-1] Livox-SDK init success!
[livox_ros2_driver_node-1] Lidar[3GGDJA600100601] status_code[0] working state[1] feature[0]
[livox_ros2_driver_node-1] Lidar start sample success
```

> `Lidar start sample success` 出现即表示雷达正常工作、正在发送点云数据。

也可用 rviz 可视化查看：

```bash
ros2 launch livox_ros2_driver livox_lidar_rviz_launch.py
```

或在终端确认有数据：

```bash
ros2 topic echo /livox/lidar --once
```

### 3.3 SDK 核心 API 一览

从 `sp_vision_25` 已有封装中提取的核心接口，写节点时按这个顺序调用：

| 步骤 | SDK 函数 | 说明 |
|------|---------|------|
| 1. 枚举设备 | `MV_CC_EnumDevices(MV_USB_DEVICE, &list)` | 查找连接到电脑的相机 |
| 2. 创建句柄 | `MV_CC_CreateHandle(&handle, pDeviceInfo)` | 创建相机操作句柄 |
| 3. 打开设备 | `MV_CC_OpenDevice(handle)` | 与相机建立连接 |
| 4. 设置参数 | `MV_CC_SetFloatValue(handle, name, val)` | 曝光时间、增益等 |
| 5. 设置参数 | `MV_CC_SetEnumValue(handle, name, val)` | 曝光模式、白平衡等 |
| 6. 设置帧率 | `MV_CC_SetFrameRate(handle, fps)` | 限制最大帧率 |
| 7. 开始取流 | `MV_CC_StartGrabbing(handle)` | 相机开始发送图像数据 |
| 8. 获取图像 | `MV_CC_GetImageBuffer(handle, &frame, timeout)` | 拿到一帧图像（轮询） |
| 9. 释放缓冲区 | `MV_CC_FreeImageBuffer(handle, &frame)` | 用完释放（必须配对调用） |
| 10. 停止取流 | `MV_CC_StopGrabbing(handle)` | 暂停图像采集 |
| 11. 关闭设备 | `MV_CC_CloseDevice(handle)` | 断开连接 |
| 12. 销毁句柄 | `MV_CC_DestroyHandle(handle)` | 清理资源 |

> USB 相机枚举时参数传 `MV_USB_DEVICE`；如果是网口相机则传 `MV_GIGE_DEVICE`。

另需注意图像格式转换：
- 海康输出的原始格式通常是 Bayer（如 `PixelType_Gvsp_BayerRG8`）
- 需要用 OpenCV 的 `cv::cvtColor()` 转为 BGR/RGB（参考 `sp_vision_25` 封装的转换映射表）

### 3.7 节点代码结构（参考 sp_vision_25 封装）

核心步骤在节点中完成：

```cpp
#include <rclcpp/rclcpp.hpp>
#include <rclcpp_components/register_node_macro.hpp>
#include <sensor_msgs/msg/image.hpp>
#include "MvCameraControl.h"

namespace tdt_vision {

class HikCameraNode : public rclcpp::Node {
public:
    explicit HikCameraNode(const rclcpp::NodeOptions& options)
        : Node("hik_camera_node", options) {
        publisher_ = create_publisher<sensor_msgs::msg::Image>("camera_image", 10);

        // 1. 枚举相机
        MV_CC_DEVICE_INFO_LIST deviceList;
        MV_CC_EnumDevices(MV_USB_DEVICE, &deviceList);

        // 2. 创建句柄 + 打开设备
        MV_CC_CreateHandle(&handle_, deviceList.pDeviceInfo[0]);
        MV_CC_OpenDevice(handle_);

        // 3. 设置参数（曝光、增益等）
        MV_CC_SetFloatValue(handle_, "ExposureTime", 5000);  // 5000us
        MV_CC_SetFrameRate(handle_, 30);

        // 4. 开始取流
        MV_CC_StartGrabbing(handle_);

        // 5. 定时器循环取图
        timer_ = create_wall_timer(std::chrono::milliseconds(33),
            std::bind(&HikCameraNode::grabAndPublish, this));
    }

private:
    void grabAndPublish() {
        MV_FRAME_OUT frame;
        if (MV_CC_GetImageBuffer(handle_, &frame, 100) != MV_OK) return;

        // frame.pBufAddr → sensor_msgs::Image → publish
        auto msg = sensor_msgs::msg::Image();
        msg.header.stamp = now();
        msg.height = frame.stFrameInfo.nHeight;
        msg.width = frame.stFrameInfo.nWidth;
        msg.encoding = "bgr8";
        msg.step = frame.stFrameInfo.nWidth * 3;
        msg.data.assign(
            (uint8_t*)frame.pBufAddr,
            (uint8_t*)frame.pBufAddr + frame.stFrameInfo.nFrameLen);

        publisher_->publish(msg);
        MV_CC_FreeImageBuffer(handle_, &frame);
    }

    void* handle_ = nullptr;
    rclcpp::Publisher<sensor_msgs::msg::Image>::SharedPtr publisher_;
    rclcpp::TimerBase::SharedPtr timer_;
};

}  // namespace tdt_vision
RCLCPP_COMPONENTS_REGISTER_NODE(tdt_vision::HikCameraNode)
```

> sp_vision_25 的封装还包含了自动 USB 重连、Bayer 格式转换等，建议直接参考它的完整实现。

```bash

```

tmux 快捷键说明：

## 六、首次部署必须做的初始化

### 6.1 录制静态场景地图

### 6.2 提取 PCD 文件

从 rosbag 中提取点云保存为 PCD，放到 `config/field_map.pcd`，然后修改以下文件中的 PCD 路径：

| 文件 | 修改位置 | 内容 |
|------|---------|------|
| `src/lidar/dynamic_cloud/src/dynamic_cloud.cpp` | 第 21 行 | `loadPCDFile("config/RM2025.pcd"` → `config/field_map.pcd` |
| `src/lidar/localization/src/localization.cpp` | 第 33 行 | `target_pcd_file = "config/RM2025.pcd"` → `config/field_map.pcd` |

> 两个 .cpp 文件中的 PCD 路径都是相对于 `ros2 launch` 执行时的工作目录，建议使用绝对路径或确认启动目录与 `config/` 的相对关系。

## 八、常见问题

| 现象 | 可能原因 |
|------|---------|
| Livox 连不上 | 广播码没改，或电脑 IP 不在 192.168.1.XXX 网段 |
| 雷达有数据但动态点云是乱的 | PCD 地图和新场景不匹配，需要重新录制 |
| launch 启动报错找不到插件 | camera_node 占位未实现，先删掉再启动 |
| 相机无数据 | 话题名不是 `camera_image`，或消息类型不是 `sensor_msgs/Image` |
| rosbag 测试正常但真实硬件不工作 | 确认各话题名和消息类型与代码中的一致 |
| `radar.launch.py` 启动报错找不到插件 | `databag_tool::BagRecorderNode` 不存在，在 launch 文件中注释掉（见 5.1） |
| `foxglove_bridge` 找不到 | `sudo apt install ros-humble-foxglove-bridge`，或在 launch 中注释掉 |
