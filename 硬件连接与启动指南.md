# 硬件连接与启动指南

## 一、项目概况

RoboMaster 2025 T-DT 雷达系统，融合 Livox Avia 激光雷达 + 海康 USB 相机，检测并跟踪场上机器人。

| 硬件 | 状态 | 说明 |
|------|------|------|
| Livox Avia 雷达 | 可连接 | 当前加载的是 RM2025 PCD 地图，换场地需重新录制 |
| 海康 USB 相机 | 可直接用 | 独立包 `hik_camera`，SDK 安装在 `/opt/MVS/` |
| rosbag 回放 | 可完整跑通 | `rosbag/全明星赛第一局bag.zip`，先解压再用 |

### 外部依赖

```bash
sudo apt install ros-humble-foxglove-bridge   # 可选，可视化调试用
sudo apt install tmux                          # 可选，一键启动用
```

## 二、激光雷达（Livox Avia）

### 2.1 修改广播码

每个雷达有唯一的广播码，需同步修改两处：

**① launch 文件** — `src/livox_driver/livox_ros2_driver/launch/livox_lidar_launch.py` 第 16 行：

```python
cmdline_bd_code = 'livox你的广播码'
```

**② 配置文件** — `src/livox_driver/livox_ros2_driver/config/livox_lidar_config.json`：

```json
"broadcast_code": "livox你的广播码",
"enable_connect": true,
```

> 两处广播码必须一致。其他 livox launch 文件（`livox_template_launch.py`、`livox_lidar_msg_launch.py` 等）也建议统一修改。

### 2.2 网络配置

雷达通过网线直连电脑。有线网卡名 `enp3s0`，设置静态 IP：

```
IP：192.168.1.100    子网掩码：255.255.255.0    网关：192.168.1.1
```

桌面设置：**设置 → 网络 → 有线 → 齿轮 → IPv4 → 手动**，填入上述值。

验证：`ip a` 确认 `enp3s0` 状态为 `UP`，有配置的 IP。

### 2.3 单独测试雷达

```bash
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 launch livox_ros2_driver livox_lidar_launch.py
```

成功标志：出现 `Lidar start sample success`。

```bash
# 终端确认有数据
ros2 topic echo /livox/lidar --once

# 或用 rviz2 查看（Fixed Frame 手动输入 livox_frame）
rviz2
```

## 五、启动流程

### 5.1 rosbag 测试（推荐先跑这个）

解压数据：
```bash
cd rosbag
unzip 全明星赛第一局bag.zip
```

启动（3 个终端，每个都先 source）：
```bash
cd ~/Ubuntu/code/rm/radar/T-DT_Radar
source /opt/ros/humble/setup.bash
source install/setup.bash
```

| 终端 | 命令 | 说明 |
|------|------|------|
| 1 | `ros2 launch dynamic_cloud lidar.launch.py` | 雷达管线 |
| 2 | `ros2 launch tdt_vision radar.launch.py` | 视觉管线（含相机+检测+解算+地图） |
| 3 | `ros2 bag play <解压后rosbag目录>` | 播放数据 |

> rosbag 中有 `/camera_image` 和 `/livox/lidar` 两个 topic，会替代真实传感器驱动。

### 5.2 真实硬件完整启动

每个终端先 source：
```bash
cd ~/Ubuntu/code/rm/radar/T-DT_Radar
source /opt/ros/humble/setup.bash
source install/setup.bash
```

| 终端 | 命令 | 说明 |
|------|------|------|
| 1 | `ros2 launch livox_ros2_driver livox_lidar_launch.py` | Livox 雷达驱动 |
| 2 | `ros2 launch dynamic_cloud lidar.launch.py` | 雷达管线（定位→动态点云→聚类→卡尔曼融合） |
| 3 | `ros2 launch tdt_vision radar.launch.py` | 视觉管线（海康相机+检测+解算+地图服务） |
| 4 | `ros2 run debug_map debug_map` | 小地图可视化（可选） |
| 5 | `rviz2` 或 `ros2 run rqt_image_view rqt_image_view` | 可视化（可选） |

> `radar.launch.py` 内部已包含海康相机节点，不需要单独 `ros2 run hik_camera`。

### 5.3 tmux 一键启动（雷达驱动+管线）

```bash
./startup.sh
```

tmux 快捷键：`Ctrl+B` + 数字切换窗口 | `Ctrl+B` + `d` 后台运行 | `tmux attach -t radar` 重新连接

## 六、话题数据流向

```
Livox 雷达 → /livox/lidar
                ↓
          localization → TF (rm_frame ↔ livox_frame)
                ↓
         dynamic_cloud → /livox/lidar_dynamic
                ↓
           cluster → /livox/lidar_cluster
                ↓
海康相机 → /camera_image → detect → /resolve_result
                                        ↓
                               kalman_filter ← /livox/lidar_cluster
                                        ↓
                              /kalman_detect → debug_map → /Radar2Sentry → 外部系统
                                        ↓
                              /radar_warn → 决策层
```

核心话题：

| 话题 | 类型 | 发布者 | 订阅者 |
|------|------|--------|--------|
| `/camera_image` | sensor_msgs/Image | hik_camera | detect |
| `/livox/lidar` | sensor_msgs/PointCloud2 | livox_driver | localization, dynamic_cloud |
| `/resolve_result` | vision_interface/DetectResult | detect→resolve | kalman_filter, debug_map |
| `/kalman_detect` | vision_interface/DetectResult | kalman_filter | debug_map |
| `/Radar2Sentry` | vision_interface/Radar2Sentry | debug_map | 外部系统 |
| `/radar_warn` | vision_interface/RadarWarn | kalman_filter | 决策层 |
| `/match_info` | vision_interface/MatchInfo | 外部系统 | resolve, kalman_filter, debug_map |

## 七、首次部署（换场地必须做）

### 7.1 录制静态场景点云

清场后（场地内无运动物体）：

```bash
# 终端1：启动雷达
ros2 launch livox_ros2_driver livox_lidar_launch.py

# 终端2：录制 10~20 秒后 Ctrl+C
ros2 bag record /livox/lidar -o ./rosbag/field_map
```

### 7.2 生成 PCD 地图

从 rosbag 提取点云 → 用 CloudCompare 处理 → 保存为 `config/field_map.pcd`，然后修改代码中的 PCD 路径：

| 文件 | 位置 | 改为 |
|------|------|------|
| `src/lidar/dynamic_cloud/src/dynamic_cloud.cpp` | 第 21 行附近 | `config/field_map.pcd` |
| `src/lidar/localization/src/localization.cpp` | 第 33 行附近 | `config/field_map.pcd` |

> 参考 `config/scripts/` 下的 `bag_to_pcd.py` 和 `CloudCompare处理指南.md`。

### 7.3 相机外参标定

`calibrate_radar.launch.py` 当前引用了不存在的 `tdt_vision::NodeCamera`，需要修复后才能用。修复方法——改为和 `radar.launch.py` 一样的结构，相机独立进程启动：

```python
# src/tdt_vision/launch/calibrate_radar.launch.py

from launch_ros.descriptions import ComposableNode
from launch_ros.actions import ComposableNodeContainer, Node
from launch import LaunchDescription

def generate_launch_description():
    hik_camera_node = Node(
        package='hik_camera',
        executable='hik_camera_node',
        name='hik_camera_node',
        output='both',
    )

    calib_container = ComposableNodeContainer(
        name='calib_container',
        namespace='',
        package='rclcpp_components',
        executable='component_container',
        composable_node_descriptions=[
            ComposableNode(
                package='tdt_vision',
                plugin='tdt_radar::Calibrate',
                name='radar_calib_node',
                extra_arguments=[{'use_intra_process_comms': True}],
            ),
        ],
        output='both',
    )

    return LaunchDescription([hik_camera_node, calib_container])
```

启动：
```bash
ros2 launch tdt_vision calibrate_radar.launch.py
```

操作：按 **Enter** → 鼠标点 5 个特征点（我方基地→我方前哨→敌方基地→敌方前哨→敌方高地）→ **WASD** 微调 → **n** 确认 → 自动写入 `config/out_matrix.yaml`。

5 个特征点的 3D 坐标在 `config/radar_config.yaml` 的 `calibrate_points` 中配置。

### 7.4 修改配置文件

换场地时需修改 `config/radar_config.yaml` 中的参数：

| 参数 | 说明 |
|------|------|
| `field_width` / `field_height` | 场地尺寸（RM 标准赛场 28×15m） |
| `perspective_points` | 透视变换参考点（相机视野四角对应的场地坐标） |
| `calibrate_points` | 标定用的 5 个特征点 3D 坐标 |
| `elevation` | 高程图路径和高度范围 |

同时确认 `config/camera_params.yaml` 中的相机内参和畸变系数是否正确。

## 九、常见问题

| 现象 | 可能原因 |
|------|---------|
| Livox 连不上 | 广播码没改，或电脑 IP 不在 192.168.1.XXX 网段 |
| 雷达有数据但动态点云是乱的 | PCD 地图和当前场景不匹配，需重新录制 |
| 相机无数据 | 先用 `/opt/MVS/bin/MVS` 确认硬件正常；检查 USB 连接 |
| rviz2 看不到点云 | Fixed Frame 下拉框手动输入 `livox_frame`（驱动不发 TF） |
| `radar.launch.py` 报错找不到插件 | 确认已注释掉不存在的节点（foxglove/record 等） |
| `foxglove_bridge` 找不到 | `sudo apt install ros-humble-foxglove-bridge`，或在 launch 中注释掉 |
| rosbag 正常但真实硬件不工作 | 检查 QoS 是否匹配（相机默认 Reliable，detect 订阅用 SensorDataQoS） |
| 标定启动报错 `NodeCamera` 不存在 | 见 7.3 修复 calibrate_radar.launch.py |
