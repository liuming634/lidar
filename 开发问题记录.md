## 1. libusb 符号冲突

### 错误信息
```
libpcl_io.so: undefined symbol: libusb_set_option
```

### 原因
- 海康相机 SDK（MVS）安装在 `/opt/MVS/`，自带了旧版 `libusb-1.0.so.0`
- `LD_LIBRARY_PATH` 中 `/opt/MVS/lib/64` 排在系统库路径前面
- 导致 PCL 加载时优先找到 MVS 自带的 libusb，缺少 `libusb_set_option` 符号

### 解决
在 `~/.profile` 中，将系统 libusb 路径放在 MVS 路径之前：

```bash
# 修改前
export LD_LIBRARY_PATH=/opt/MVS/lib/64:/opt/MVS/lib/32:$LD_LIBRARY_PATH

# 修改后
export LD_LIBRARY_PATH=/usr/lib/x86_64-linux-gnu:/opt/MVS/lib/64:/opt/MVS/lib/32:$LD_LIBRARY_PATH
```

系统 libusb 版本 1.0.25，兼容 MVS SDK。

## 2. cv::FileStorage 读取 YAML 崩溃

### 错误信息
```
OpenCV(4.5.4) ... Input file is invalid in function 'open'
Component constructor threw an exception: ...
```

### 原因
OpenCV 的 `cv::FileStorage` 在无法打开文件时会抛出 `cv::Exception`，节点构造函数中未捕获异常导致整个节点崩溃。

### 解决
1. **去掉 YAML 行内注释**：`cv::FileStorage` 的 YAML 解析器不支持行内注释（如 `field_width: 28.0 # 注释`），会解析失败。注释必须单独成行。
2. **添加 try-catch 保护**：所有 `cv::FileStorage` 读取操作包在 try-catch 中，失败时使用默认值。

```cpp
try {
    cv::FileStorage fs("./config/params/field_params.yaml", cv::FileStorage::READ);
    if (fs.isOpened()) {
        fs["field_width"] >> field_width_;
        fs["field_height"] >> field_height_;
        fs.release();
    }
} catch (const cv::Exception& e) {
    RCLCPP_WARN(this->get_logger(), "Failed to read config, use defaults: %s", e.what());
}
```

### 涉及节点
- `localization` → `config/params/gicp_crop.yaml`
- `dynamic_cloud` → `config/params/dynamic_crop.yaml`
- `kalman_filter` → `config/params/field_params.yaml`
- `resolve` → `config/params/field_params.yaml`
- `debug_map` → `config/params/field_params.yaml`

## 3. 构建设置提醒

### symlink-install
```bash
colcon build --symlink-install --cmake-args -DCMAKE_BUILD_TYPE=Release -DCMAKE_EXPORT_COMPILE_COMMANDS=1
```

使用 `--symlink-install` 可以让 install 目录中的文件链接回源文件，修改源码后不需要重新 build 即可测试（仅头文件修改或 Python 节点有效，C++ 仍需要编译）。
